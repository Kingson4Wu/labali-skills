# Good Commit Message Examples

These examples demonstrate the target quality for commit messages generated by this skill.

---

## Example 1 — Feature with clear scope

```
feat(skill): add license and compatibility fields to all SKILL.md frontmatter

- add license: MIT to all 17 skills derived from root LICENSE
- add compatibility per skill type: macOS-only for Vision OCR skills,
  macOS/Linux + Chrome CDP for browser skills, AI-only for policy skills
- correct labali-image-desensitize compatibility to ImageMagick (not ffmpeg)
- correct labali-spotify-* to require agent-browser CLI in PATH
```

**Why it works:**
- Subject is specific and action-oriented, under 72 chars
- Body bullets map to real grouped changes, not generic "update files"
- Scope `(skill)` accurately signals the affected area

---

## Example 2 — Docs restructure

```
docs: restructure standards into docs/ and unify AI entry points

- move DEVELOPMENT.md sections into docs/skill-authoring.md,
  docs/conventions.md, docs/testing.md, docs/workflow.md
- replace CLAUDE.md body with pointer to AGENTS.md
- add Quick Links table in AGENTS.md for per-task doc loading
- keep DEVELOPMENT.md as overview only
```

**Why it works:**
- Type `docs` is correct for non-code changes
- Body explains the reorganization by action, not by file list
- "keep DEVELOPMENT.md as overview only" captures the constraint decision

## Example 3 — Bug fix with concise body

```
fix(browser): handle stale CDP connection on cold Chrome start

- detect empty tab list on connect and retry with 2s delay (max 3 times)
- surface actionable error when port is open but Chrome is not ready
- add test fixture for cold-start scenario
```

**Why it works:**
- Subject names the exact symptom, not just "fix bug"
- Body describes mechanism + error surface + test, no padding
- Under 72 chars in subject

---

## Example 4 — Chore / housekeeping

```
chore: standardize skill structure and enforce Chinese language gate

- add missing README.md and README.zh-CN.md to 6 skills
- add .gitignore entry for .cache/ in browser skills
- enforce no-Chinese rule via check:chinese npm script
- add skills:validate script for post-change CI gate
```

**Why it works:**
- `chore` type is correct (no production behavior change)
- Groups related housekeeping into 4 concrete bullet points
- Avoids vague "clean up", names what was actually done
