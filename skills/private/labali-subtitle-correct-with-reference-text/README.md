# labali-subtitle-correct-with-reference-text Usage

This skill corrects subtitle wording by aligning subtitle cues to a reference script that has accurate text but no timestamps. Timestamps in the original subtitle are preserved unchanged.

## 1) Install and Update

Install from GitHub:

```bash
npx skills add github.com/Kingson4Wu/labali-skills --skill labali-subtitle-correct-with-reference-text
```

Update to latest version: run the install command again.

## 2) Quick Start

```bash
npx tsx skills/private/labali-subtitle-correct-with-reference-text/scripts/run.ts \
  --subtitle_path "/path/to/input.srt" \
  --reference_path "/path/to/reference.txt"
```

Optional flag:
- `--output_path "/path/to/output.srt"` — defaults to `<subtitle_stem>.corrected.<ext>` in the same directory

Supports both `.srt` and `.vtt` subtitle formats.

## 3) What It Does

1. Parses the input subtitle file into timed cues.
2. Reads the reference text file (plain text, no timestamps required).
3. Aligns cue sequences to reference text by sequence matching.
4. Replaces each cue's text with the aligned reference segment.
5. Writes the corrected subtitle file with all original timestamps preserved.

## 4) Typical Use Case

Use this skill when you have:
- A subtitle generated by Whisper or similar ASR tool (correct timing, but word errors)
- A manually corrected transcript or reference script (correct wording, no timestamps)

The skill merges the timing from the subtitle with the text from the reference.

## 5) Limitations

1. **Cue count must roughly match** — alignment works best when the number of subtitle cues is close to the number of reference text segments; large structural mismatches reduce accuracy.
2. **No external API calls** — fully offline; all alignment is done locally.
3. **Format support** — `.srt` and `.vtt` only.

## 6) Troubleshooting

- Output text looks misaligned: check that the reference text covers the same content as the subtitle and has similar segment granularity.
- `FileNotFoundError`: confirm both `--subtitle_path` and `--reference_path` point to existing files.
