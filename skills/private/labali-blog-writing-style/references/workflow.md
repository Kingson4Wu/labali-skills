# Workflow

Use the matching workflow based on the task type.

## 1. New Article Workflow

1. Determine the subtype:
   - technical summary,
   - troubleshooting,
   - workplace reflection,
   - reading note / idea note.
2. Clarify the central claim in one sentence.
3. Build a simple article skeleton:
   - intro / trigger,
   - main analysis sections,
   - practical judgment or closing summary.
4. Draft in the author's voice:
   - pragmatic,
   - direct,
   - reasoned,
   - with clear transitions.
5. Tighten the structure:
   - merge redundant bullets into paragraphs,
   - promote key logical chunks into headings,
   - keep lists only where lists truly help.
6. Read once for "AI drift":
   - remove vague abstraction,
   - remove generic uplift,
   - replace empty claims with concrete reasoning.

## 2. Rewrite Existing Article Workflow

1. Read the article fully before changing tone.
2. Identify what must not change:
   - core meaning,
   - key claims,
   - central examples,
   - intended emotional stance.
3. Diagnose the main quality issues:
   - weak hierarchy,
   - too many bullets,
   - repetitive transitions,
   - overly abstract phrasing,
   - AI tone drift.
4. Rewrite in this order:
   - structure,
   - transitions,
   - paragraph shape,
   - phrasing polish.
5. Preserve the author's judgment style while improving article readability.
6. If a section is list-heavy but conceptually linear, convert it into prose.
7. If a section contains true categories, keep a list but make each item substantial.

## 3. Partial Refinement Workflow

Use when only some paragraphs or sections are being revised.

1. Identify the local section's role in the whole article.
2. Keep tone and logic consistent with surrounding sections.
3. Improve:
   - local hierarchy,
   - sentence flow,
   - clarity,
   - voice consistency.
4. Avoid introducing style mismatch between revised and unrevised sections.

## 4. Structure Rules

- Prefer article-shaped prose over note-shaped stacking.
- Use `##` for major sections and `###` only when a true subdivision exists.
- Avoid one-line subsections unless necessary.
- Use lists for:
  - steps,
  - comparisons,
  - grouped categories,
  - explicit tradeoff tables in prose/list form.
- Avoid lists for simple emotional or conceptual exposition that reads better as paragraphs.

## 5. Revision Checklist

Before finishing, verify:

- The core meaning is preserved.
- The text sounds like the author, not a generic AI explainer.
- The structure is clearer than older posts.
- Lists are selective, not the default.
- There is enough transition between sections.
- The ending lands on experience, judgment, or reminder rather than forced inspiration.
