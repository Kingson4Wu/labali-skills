## General Principles

When asked for a solution, start with the simplest approach (e.g., a symlink, a single command, a single file) before proposing complex architectures. Do not over-engineer.

## Writing & Documentation

When summarizing or compressing content, preserve all actionable details and examples. Ask before removing sections. Default to more detail, not less.
